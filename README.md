# Visita Chada Valencia Tourism Website

A responsive tourism website for Valencia, Negros Oriental. This project presents local destinations, travel information, community updates, delicacies, transportation details, and the developer team behind the website.

## Project Description

I created this website to promote Valencia as a travel destination and to give visitors a simple way to explore what the municipality offers. The website focuses on clean navigation, visual tourism content, responsive layouts, and an interactive developers page.

The project was improved to make the code cleaner, faster, and easier to maintain. Images were optimized, the layout was made responsive, and the interactive parts were rewritten using better HTML, CSS, and JavaScript practices.

## Features

- Responsive homepage for Valencia tourism information
- Latest news and event highlights
- About and location sections
- Travel planning page with destinations, accommodations, food, and transportation
- Interactive developers page with expandable team profile cards
- Mobile-friendly navigation menu
- Optimized WebP images for faster loading
- Clean semantic HTML structure
- Organized CSS and JavaScript files

## Pages

- `index.html` - Homepage with hero section, news, about, and location information
- `pages/travel.html` - Travel guide page for destinations, food, accommodation, and transportation
- `pages/us.html` - Developers page with interactive team cards

## How to Run the Website

You can open the website directly by double-clicking `index.html`.

For a better local preview, run a simple local server:

```bash
python -m http.server 8000
```

Then open this in your browser:

```text
http://localhost:8000
```

## Improvements Made

I improved the project by focusing on performance, responsiveness, accessibility, and code organization.

- Replaced heavy media usage with optimized images
- Used reusable CSS classes for sections, cards, buttons, and layouts
- Improved mobile responsiveness using flexible grids and media queries
- Added proper semantic tags such as `header`, `nav`, `main`, `section`, `article`, and `footer`
- Reworked JavaScript interactions using event listeners instead of inline `onclick`
- Improved the developers page while keeping the original interactive card concept
- Added better image alt text and navigation accessibility
